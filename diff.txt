+  - Set this to `true` to enable the AI vector generation using the `clip-ViT-B-32` model.
+  - **Important Database Requirement**: If enabled, your CMS database target table (e.g., `Photo`) **MUST** have an `imageVector` column of type `vector` (via the `pgvector` extension). If the column is missing, the SQL update will crash.
+  - **Memory Requirement**: Loading the `clip-ViT-B-32` model consumes approximately 1GB~1.5GB of RAM per worker. If enabled, ensure your deployment environment (e.g., Cloud Run or VM) has sufficient memory (recommended: 8GB - 16GB, depending on your uWSGI worker settings).
